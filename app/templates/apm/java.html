OneAPM for Java

OneAPM for Java 能够深入到所有 Java 应用内部完成应用性能管理和监控，包括代码级别性能问题的可见性、性能瓶颈的快速识别与追溯、真实用户体验监控、服务器监控和端到端的应用性能管理。

而这一切，只需将 OneAPM Jar 包添加至应用服务器中。OneAPM 的部署简易性，让你即刻启程！

安装指南

总览全局性能

应用首页的应用性能总览，能够让你迅速感知应用的健康程度和总体性能状态与问题。以总览为入口，可以轻松追溯至地理信息、关键事务、JVM 性能、错误统计等性能度量点，完成应用性能管理工作。在应用性能总览中，包括以下信息：

  • 响应时间
  • 性能指数
  • 吞吐量（单位：rpm 每分钟请求数）
  • 关键事务
  • 错误率
  • 近期事件
  • 服务器信息

快速定位瓶颈

按照浏览器和地理信息 2 种分类方式，对真实用户的页面加载时间等性能数据进行查看，快速区隔出需要优化性能的浏览器类型和区域用户。

除开真实用户体验监控以外，OneAPM for Java 可以达到代码级别的性能可见性，如 SQL 语句的性能分析和问题定位。

关键事务 Top 5，平均响应时间、吞吐量这些性能查看方式，帮助你快速定位性能瓶颈；事务的深度 Traces 记录，协助你完成应用性能管理从抽象到具象的追溯过程。

  • 使用跨应用追踪功能和应用拓扑建模功能，完成分布式应用和 SOA 应用的应用性能问题追溯
  • 比较不同版本的应用性能
  • 同时监控后端事务和前端事务
  • 通过报警机制，确保在性能问题造成损失前，完成修复工作

深度追溯细节

OneAPM 提供整齐划一的界面，让你能够无障碍地完成从真实用户体验，到应用服务器，直至应用代码的端到端应用性能管理。

OneAPM 支持团队协助，能够帮助一个团队完成应用性能管理工作，包括：事务的深度性能剖析、浏览器 Traces 记录查看、关键事务性能分析、事务 Traces 记录查看。

浏览器 Traces 记录和事务 Traces 的连通性，确保你可以完成 针对特殊交互的端到端性能追踪。

详悉错误报告

直观的错误率和错误报告中的性能细节，从宏观和细节 2 个角度展示终端用户的真实体验。其中，错误细节包括：错误 URL 地址、HTTP 状态码、错误次数、用户得到的错误消息。

通过设置错误率阈值，来触发报警，确保你可以随时随地监控应用性能。默认触发报警的 HTTP 状态码，如右所示：

  • 400 Bad Request
  • 401 Unauthorized
  • 403 Forbidden
  • 502 Bad Gateway
  • 503 Service Unavailable

SQL 性能分析

OneAPM 通过 JDBC 请求来度量数据库操作的性能表现。支持的数据库包括：MySQL、Postgres、Oracle、jTDS (SQL Server 和 Sybase) 、sqljdbc4 (SQL Server) 和 Apache Derby。

OneAPM 从数据聚合、事务崩溃、后台任务和事务 Traces 记录 4 个角度，来度量数据库请求的性能表现。

对于性能表现差的事务，OneAPM 提供详细的 SQL 请求信息。其中详细信息包括：请求时长和请求数量，以及附加的 SQL 参数信息。在 MySQL 和 Postgres 数据库中，OneAPM 提供每条 SQL 语句的详细解释。

综上所有特性，OneAPM 能够为你提供数据库活动的全局信息，并协助你找到性能优化点。

外部 HTTP 请求

对于很多应用来说，对事务响应时间影响最大的是外部HTTP请求。外部请求包括发往系统内部的请求，或者发往外部服务（如 Amazon S3 等）的请求。

在应用性能总览、应用拓扑和事务 Traces 记录中，都可以查看到外部 HTTP 请求的性能表现记录。通过配置 OneAPM，你可以选择查看全部或部分 HTTP 请求参数。

如果你同时监控多个应用，OneAPM 提供跨应用的外部 HTTP 请求性能监控；在事务的总览页面、应用拓扑页面和事务 Traces 页面中，你可以查看多个事务之间的关联关系。

JVM 性能监控

OneAPM 可以监控 JVM 和应用服务器的性能，从而对性能排查、性能调优、和基础设施建设工作起到指导和协助的作用。不同的应用服务器，我们可以不同程度地监控以下性能相关信息：

  • 内存分配情况 (heap & non-heap areas)
  • 垃圾回收情况
  • JVM: 类加载/卸载数
  • JVM: 活跃线程数
  • 线程池：活跃/闲置线程数
  • 线程池：活跃线程百分比
  • 应用：活跃/已过期/已拒绝 HTTP 会话数
  • 应用服务器：活跃/已完成/已创建事务数

如何安装 OneAPM ？

很开心听到这个问题！
以下是 Java Agent 安装指南。具体安装过程可能会因为 IDE 环境不同而有所偏差。

1. 创建 OneAPM 账号

2. 下载 OneAPM Java Agent

oneapm_agent.zip

3. 安装于你的环境

Linux or Mac

解压文件至应用主目录中：

unzip oneapm_agent.zip –d /path/to/appserver/

在 oneapm 目录中运行安装指令：

cd /path/to/appserver/oneapm
java –jar oneapm.jar install

Windows

打开 oneapm_agent.zip
复制 oneapm 文件夹至你的应用服务器主目录下
打开 CMD，并定位至应用服务器主目录下
在 oneapm 目录下，执行安装命令

cd oneapm
java –jar oneapm.jar install

4. 重启应用

几分钟后，你的应用就会开始向 OneAPM 发送数据。
我们收到数据后，你的应用就会列在 OneAPM 应用列表中。
