OneAPM for .NET 能够深入到所有 .NET 应用内部完成应用性能管理和监控，包括代码级别性能问题的可见性、性能瓶颈的快速识别与追溯、真实用户体验监控、服务器监控和端到端的应用性能管理。

端到端的性能可见性

按照浏览器和地理信息 2 种分类方式，对真实用户的页面加载时间等性能数据进行查看，快速区隔出需要优化性能的浏览器类型和区域用户。

除开真实用户体验监控以外，OneAPM for .NET 可以达到代码级别的性能可见性，如 SQL 语句的性能分析和问题定位；还可以查看各组件对应用性能的影响，确保在用户意识到性能问题前，完成修复工作。

  • 使用跨应用追踪功能和应用拓扑建模功能，完成分布式应用和 SOA 应用的应用性能问题追溯
  • 比较不同版本的应用性能
  • 通过对吞吐量和集群的性能优化，来避免硬件成本消耗
  • 通过报警机制，确保在性能问题造成损失前，完成修复工作
  • 详悉线程信息，即刻开始性能问题修复

支持主流框架/平台

OneAPM支持所有 .NET 语系： VB.NET、C# 和 C++/CLI for .NET 2.0 及以上版本。OneAPM 支持主要框架：MVC 2、MVC3、MVC4、ASPs 和 WebServices。

除开真实用户体验监控以外，OneAPM for .NET 可以达到代码级别的性能可见性，如 SQL 语句的性能分析和问题定位。

OneAPM 可以监测私有数据中心中的应用，和部署于公共云上的应用，包括：Microsoft Azure 和 Azure Cloud Services。

协同合作，只在 OneAPM

OneAPM 给团队中每个角色分配不同权限，管理应用性能，协同合作确保应用长期处于健康稳定状态。通过设置报警触发器和通知方案，确保在错误发生和性能出现问题时，能够通知相关人员进行修复；健全的 Traces 记录能够帮助相关人员快速定位性能瓶颈。
