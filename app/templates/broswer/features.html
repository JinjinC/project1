围绕用户体验

OneAPM 从地理位置、浏览器类型、设备类型 3 个类别，来划分真实用户体验； 方便你从不同角度查看应用的性能表现

OneAPM 划分出性能表现差的具体范围，指导性能修复工作，从而减少工作量并提高效率。

更广视野

页面 Trace 给你更广视野，针对不同 URL 建立页面的 Trace，记录 TCP、DNS、DOM 加载、渲染等时间， 让你能够了解整个加载过程。

页面 Trace 就是一次前端页面加载的快照，快照信息具体到：浏览器类型、IP 地址等，并包括 TCP 传输性能和 DNS 时间。

归类错误

从错误类型、URL、浏览器 3 个角度，归类错误的发生。

OneAPM 统计错误发生的次数，并可以跟踪到单次错误的 URL、发生时间，和与之关联的浏览器类型；以及更为底层的信息：错误信息、相关参数和 Stack Trace。

深入至 AJAX

单页面 Web 应用往往通过 AJAX 进行前后端交互，所以「页面 trace」不再适用于 AJAX 性能监控。

OneAPM 针对 AJAX 性能，特别增设响应时间、数据请求和接收、HTTP 状态的性能监控，完善 Browser Insight 功能。
